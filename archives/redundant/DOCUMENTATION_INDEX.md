## ğŸ—ï¸ Architecture Overview

### Multi-Tenant Architecture
- Tenant isolation via `tenant_id` on all tables
- Separate billing and configuration per tenant
- Shared infrastructure with data isolation
- RBAC (Role-Based Access Control)
- ABAC (Attribute-Based Access Control)

### Technology Stack
- **Backend**: Go 1.24
- **Database**: MySQL 8.0.44
- **Frontend**: Next.js 16.0.3
- **Cache**: Redis 7-alpine
- **Message Queue**: Redis (built-in)
- **Container**: Docker/Podman
- **Monitoring**: Prometheus + Grafana

### Database Design Patterns
- ULID primary keys (sortable, collision-free)
- Hierarchical structures (parent_id, level)
- JSON configuration (flexible attributes)
- Audit triggers (automatic change tracking)
- Soft deletes (compliance & recovery)
- Denormalized metrics (performance)
- Encryption at rest (sensitive data)
